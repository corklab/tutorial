GitHub Pagesでつくる無料ホスティングのLP公開

チュートリアルで使っている「GitHub Pages」について、もう少し詳しく知っておきましょう

GitHub Pagesとは何か

GitHub Pagesは、GitHubが提供している無料のWebサイト公開サービスです。

通常、Webサイトを公開するにはサーバーを借りたり、ホスティングサービスに登録したりする必要があります。GitHub Pagesなら、GitHubのリポジトリにHTMLファイルを置くだけで、そのままWebサイトとして公開されます。

ひとことで言うと

「GitHubに置いたHTMLファイルを、そのままWebサイトとして世界中に公開してくれる無料サービス」です。

公開されたサイトには、以下の形式のURLが自動で割り当てられます：

https://アカウント名.github.io/リポジトリ名/

このチュートリアルの場合は：

https://corklab.github.io/tutorial/

できること

HTML・CSS・JavaScriptのサイトを公開できる

LPやポートフォリオ、イベント告知ページなど、「見た目のあるWebページ」をそのまま公開できます。Claudeが作ったHTMLファイルをそのまま公開できるのはこの仕組みのおかげです。

無料で使える

GitHubの無料アカウントで利用できます。サーバー代やドメイン維持費は一切かかりません（独自ドメインを使う場合を除く）。

HTTPSが標準対応

公開されるサイトは最初からHTTPS（暗号化通信）に対応しています。「このサイトは安全ではありません」といった警告が出ることはありません。

Gitと連動している

リポジトリのmainブランチにマージするだけで、サイトが自動的に更新されます。チュートリアルで学んだ「Claudeに修正してもらう → PR → マージ → 反映」のサイクルがそのまま使えるのはこの仕組みです。

留意事項（気をつけること）

静的サイトのみ対応

GitHub Pagesで公開できるのは静的サイト（HTML・CSS・JavaScriptだけで動くサイト）です。

できないことの例

お問い合わせフォームの送信内容をデータベースに保存する、ユーザーのログイン機能を作る、商品の決済処理を行う、といったサーバー側の処理が必要な機能は、GitHub Pages単体では実現できません。

公開リポジトリは中身が見える

GitHub Pagesで公開するリポジトリが「Public（公開）」設定の場合、HTMLのソースコードや画像ファイルなど、リポジトリの中身は誰でも見ることができます。パスワードやAPIキーなど、秘密にしたい情報はリポジトリに入れないようにしましょう。

サイトの容量と帯域に制限がある

GitHubが推奨する目安は以下のとおりです：

項目推奨上限
サイト全体のサイズ1 GB まで
月間の転送量（帯域）100 GB まで

LPや数ページのサイトであれば、まず超えることはありません。大量の動画や高解像度画像を置く場合は注意してください。

商用利用について

GitHub Pagesは商用サイトの公開を禁止していませんが、オンラインストアのような本格的な商用サービスの運用には向いていません。LPや会社紹介ページ程度であれば問題なく利用できます。

独自ドメインの設定

GitHub Pagesでは、デフォルトの *.github.io の代わりに、自分で取得したドメイン（例：www.example.com）を使うこともできます。

設定の流れ

  1. ドメインを取得する — ドメイン登録サービス（お名前.com、Google Domains、Cloudflareなど）で好きなドメインを購入する
  2. DNS設定をする — 取得したドメインの管理画面で、GitHubのサーバーを向くようにDNSレコードを設定する
  3. GitHubで設定する — リポジトリの Settings → Pages → Custom domain に取得したドメインを入力して保存する
  4. HTTPS化 — 「Enforce HTTPS」にチェックを入れる（DNSが浸透した後に有効化できます）

DNS設定の具体例

Aレコード（ルートドメインの場合）：以下のIPアドレスを設定
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

CNAMEレコード（サブドメインの場合）：
www → corklab.github.io

ポイント

独自ドメインの設定は管理者が行います。ドメインの取得には年間数百円〜数千円の費用がかかります。まずはデフォルトの *.github.io のURLで始めて、必要になったら独自ドメインを検討するのがおすすめです。
