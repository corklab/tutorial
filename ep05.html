第5話

世界へ公開・マージしてLPを本番反映

目標：プルリクエストをマージして、LPが公開されたことを確認する

この話の流れ

あなた（ブラウザ） GitHub | | |-- PRの内容を確認 | |-- マージボタンを押す ---->| | |-- 自動で公開処理 | |-- URL発行 |-- URLにアクセス | |-- LP表示確認！ |

実践ステップ

ステップ1：プルリクエストをマージする

前の話で作ったプルリクエストのページを開きます。

  1. PRの内容を確認する（Claudeが作ったファイルが表示されています）
  2. 緑色の「Merge pull request」ボタンを押す
  3. 「Confirm merge」を押す

マージって何？

別室（ブランチ）で作ったものを、本番のテーブル（main）に移すことです。これでmainにLPのHTMLファイルが入ります。

ステップ2：公開されたURLを確認する

管理者が事前にGitHub Pagesの設定を済ませてくれているので、マージするだけでLPが自動的に公開されます。1〜2分待ってから以下のURLにアクセスしましょう：

https://corklab.github.io/tutorial/

おめでとうございます！

あなたがClaudeと一緒に作ったLPが、世界中の誰でもアクセスできるURLで公開されました！

ステップ3：PRのステータスを確認する（任意）

GitHubでプルリクエストのページを開くと、ステータスが「Merged」（紫色）に変わっているのが確認できます。

トラブルシューティング

URLにアクセスしても表示されない場合

  • マージ後、反映まで1〜2分かかることがあります。少し待ってリロードしてみてください
  • 初回公開の場合、管理者にPages設定が完了しているか確認してみてください

更新したはずなのに古い内容が表示される場合

ブラウザが以前のページをキャッシュ（一時保存）している可能性があります。以下の方法でキャッシュをクリアしてからリロードしてください。

Macでのキャッシュクリア付きリロード

Safari：Command + Option + R を押す
Google Chrome：Command + Shift + R を押す

これで、キャッシュを無視してサーバーから最新のページを取得し直します（「スーパーリロード」と呼ばれます）。

404エラーが出る場合

  • ファイル名が index.html になっているか確認してください
  • 心当たりがなければ管理者に相談しましょう

この話のまとめ

やったこと誰が
PRの確認・マージあなた（ブラウザ）
公開処理GitHub（自動）
公開URLの確認あなた（ブラウザ）

マージボタンを押すだけで、LPが世界に公開されました。Pages設定は管理者が最初に済ませてくれているので、あなたは今後マージするたびに自動で反映されます。
