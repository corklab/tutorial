第5話

世界へ公開・管理者のマージで本番反映

目標：管理者がPRをマージしてくれたら、LPが公開されたことを確認する

この話の流れ

あなた 管理者 GitHub | | | |-- PR作成済み --->| | | |-- PRを確認 | | |-- マージ実行 ---->| | |-- Pages設定 ----->|-- 公開処理 | | |-- URL発行 | |-- 「公開したよ」-->| |-- URLにアクセス | |-- LP表示確認！ |

実践ステップ

ステップ1：管理者のマージを待つ

前の話でプルリクエストを作成しました。ここからは管理者の作業です：

  1. 管理者があなたのPRの内容を確認する
  2. 問題なければ管理者がマージ（本番に反映）する
  3. 管理者がGitHub Pagesの設定を行う

待っている間は？

管理者がマージするまで少し待ちましょう。マージが完了すると、GitHubから通知メールが届きます。管理者から公開URLを教えてもらえます。

ステップ2：公開されたURLを確認する

管理者からURLを教えてもらったら、ブラウザでアクセスします：

https://corklab.github.io/tutorial/

おめでとうございます！

あなたがClaudeと一緒に作ったLPが、世界中の誰でもアクセスできるURLで公開されました！

ステップ3：PRのステータスを確認する（任意）

GitHubでプルリクエストのページを開くと、ステータスが「Merged」（紫色）に変わっているのが確認できます。

トラブルシューティング

URLにアクセスしても表示されない場合

  • マージ後、反映まで1〜2分かかることがあります。少し待ってリロードしてみてください
  • 管理者にPages設定が完了しているか確認してみてください

404エラーが出る場合

  • ファイル名が index.html になっているか確認してください
  • 心当たりがなければ管理者に相談しましょう

この話のまとめ

やったこと誰が
PRの確認・マージ管理者
Pages公開設定管理者
公開URLの確認あなた（ブラウザ）

あなたはURLを開いて確認するだけ。技術的な公開設定は管理者がすべてやってくれます。
