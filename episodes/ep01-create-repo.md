# 第1話：GitHubでリポジトリ作成とClaudeへのクローン指示

## 目標

GitHubで箱（リポジトリ）を作り、MacのClaudeに引き継ぐ。

## この話で起きること

```
あなた（ブラウザ）        Claude（Mac）
    |                        |
    |-- リポジトリを作る ------>|
    |-- URLをコピー ---------->|
    |-- 「クローンして」------->|
    |                        |-- クローン実行
    |                        |-- 作業フォルダ完成
```

---

## 実践ステップ

### ステップ1：GitHubで新しいリポジトリを作る

1. ブラウザで [github.com/new](https://github.com/new) を開く
2. 以下を入力する：
   - **Repository name**: `my-landing-page`（好きな名前でOK）
   - **Description**: 空欄でOK
   - **Public** を選択（GitHub Pagesで公開するため）
3. **READMEやファイルは何も追加しない**（空のまま）
4. 「Create repository」ボタンを押す

> **なぜ空にするの？**
> Claudeが最初からすべてのファイルを作ってくれるので、最初は何もない状態がベストです。

### ステップ2：リポジトリのURLをコピーする

作成されたリポジトリのページで、ブラウザのアドレスバーからURLをコピーします。

```
https://github.com/あなたのユーザー名/my-landing-page
```

### ステップ3：Claudeに指示を出す

MacのClaudeアプリを開いて、以下のように話しかけます：

> **あなた → Claude:**
>
> 「このGitHubリポジトリをMacにクローンして、LP作成の準備をして」
>
> https://github.com/あなたのユーザー名/my-landing-page

### ステップ4：Claudeの動きを見守る

Claudeが以下を自動で実行します：

1. Mac上の適切な場所にフォルダを作成
2. `git clone` コマンドでリポジトリをダウンロード
3. 作業フォルダの準備完了を報告

> **ポイント**: Claudeがターミナルコマンドを実行するとき、許可を求めてくることがあります。内容を確認して「Allow」を押してください。

---

## この話のまとめ

| やったこと | 誰が |
|-----------|------|
| リポジトリ作成 | あなた（ブラウザ） |
| URLコピー | あなた（ブラウザ） |
| クローン実行 | Claude（Mac） |

あなたがやったのは「ブラウザでポチポチ」と「Claudeに一言伝える」だけ。技術的な部分はすべてClaudeが代行してくれました。
