# 第6話：修正も一瞬・Claudeと回す更新サイクル

## 目標

LPの文言を修正して再公開する一連の流れをマスターする。

## この話で起きること

```
あなた（Claude画面/ブラウザ）    Claude（Mac）
    |                              |
    |-- 「見出し変えて、         -->|
    |    コミットとプッシュまで     |
    |    やっておいて」            |
    |                              |-- ブランチ作成
    |                              |-- ファイル修正
    |                              |-- コミット
    |                              |-- プッシュ
    |                              |-- 完了報告
    |                              |
    |-- ブラウザでPR作成 --------->|
    |-- マージ ------------------->|
    |-- LP更新完了 🎉              |
```

---

## 実践ステップ

### ステップ1：Claudeに修正からプッシュまで一気に任せる

Claudeに以下のように話しかけます：

> **あなた → Claude:**
>
> 「LPの見出しを『最高のコーヒーをあなたに』に変えて、コミットとプッシュまでやっておいて」

これ一言で、Claudeは以下をすべて自動でやってくれます：

1. 新しいブランチを作成
2. HTMLファイルの該当箇所を修正
3. 変更をコミット
4. GitHubにプッシュ

### ステップ2：ブラウザでプルリクエストを作成・マージする

1. GitHubでリポジトリを開く
2. 「Compare & pull request」をクリック
3. 「Create pull request」をクリック
4. 「Merge pull request」→「Confirm merge」をクリック

### ステップ3：更新を確認する

公開URLにアクセスして、見出しが変わっていることを確認します。

```
https://あなたのユーザー名.github.io/my-landing-page/
```

> **注意**: 反映まで1〜2分かかることがあります。変わらない場合はブラウザのキャッシュをクリアするか、少し待ってからリロードしてください。

## 更新サイクルまとめ

これであなたは、以下のサイクルを回せるようになりました：

```
┌─────────────────────────────────────────┐
│                                         │
│  ① Claudeに修正を指示                   │
│      ↓                                  │
│  ② Claude: 修正→コミット→プッシュ        │
│      ↓                                  │
│  ③ あなた: PR作成→マージ（ブラウザ）      │
│      ↓                                  │
│  ④ LP更新完了！                          │
│      ↓                                  │
│  （①に戻る）                             │
│                                         │
└─────────────────────────────────────────┘
```

何回でも、何度でも、このサイクルを回すだけでLPを進化させ続けられます。

## 応用：こんなことも頼めます

- 「写真を追加して」
- 「ボタンの色を赤に変えて」
- 「スマホでも見やすくして」
- 「お問い合わせフォームを追加して」
- 「全体的にもっとポップな雰囲気にして」

具体的に伝えるほど、思い通りの結果になります。
