# 第4話：Pushも丸投げ・GitHubへデータを打ち上げる

## 目標

Claudeにプッシュさせ、ブラウザでプルリクエストを作成する。

## この話で起きること

```
あなた（Claude画面/ブラウザ）    Claude（Mac）
    |                              |
    |-- 「プッシュして」---------->|
    |                              |-- push実行
    |                              |-- 完了報告
    |                              |
    |-- ブラウザでGitHubを開く ---->|
    |-- プルリクエスト作成 -------->|
```

---

## 実践ステップ

### ステップ1：Claudeにプッシュを指示する

Claudeに以下のように話しかけます：

> **あなた → Claude:**
>
> 「今のブランチをGitHubにプッシュして」

### ステップ2：Claudeがプッシュを実行する

Claudeが以下を実行します：

1. ローカル（Mac）のデータをGitHub（インターネット上）に送信
2. 完了を報告

> **プッシュって何？**
>
> Mac上にしかなかったデータを、GitHubのサーバーに「打ち上げる」ことです。
> これにより、ブラウザからでもデータが見られるようになります。

### ステップ3：ブラウザでプルリクエストを作成する

1. ブラウザで自分のリポジトリを開く
   ```
   https://github.com/あなたのユーザー名/my-landing-page
   ```

2. 黄色い帯で「Compare & pull request」ボタンが表示されているのでクリック

3. 以下を入力：
   - **タイトル**: 「LPを作成」など、自由に
   - **説明**: 空欄でOK
   
4. 「Create pull request」ボタンを押す

> **プルリクエストって何？**
>
> 「この変更を本番に入れてもいいですか？」という確認の場です。
> 今回は自分だけのプロジェクトなので儀式的ですが、チーム開発では重要なステップです。
> 次の話でこれを承認（マージ）して本番に反映します。

---

## この話のまとめ

| やったこと | 誰が |
|-----------|------|
| 「プッシュして」と伝える | あなた（Claude画面） |
| プッシュ実行 | Claude（Mac） |
| プルリクエスト作成 | あなた（ブラウザ） |

Claudeがデータを打ち上げた後、あなたがブラウザで「確認の場」を用意しました。
